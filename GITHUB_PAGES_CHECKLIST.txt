Scaffold AC — GitHub Pages (404) — tjekliste
============================================

Officiel GitHub-hjælp (404):
https://docs.github.com/en/pages/getting-started-with-github-pages/troubleshooting-404-errors-for-github-pages-sites

Kilde: “Publishing source” (branch /docs, Actions, osv.):
https://docs.github.com/en/pages/getting-started-with-github-pages/configuring-a-publishing-source-for-your-github-pages-site

1) REPO SKAL VÆRE PUBLIC (gratis GitHub)
   Private repo + gratis konto = ofte ingen/offentlig Pages-URL → 404.
   Settings → General → Danger zone → Change repository visibility → Public.

2) METODE 0 (ANBEFALET — INGEN ACTIONS)
   Dette er den indbyggede GitHub-model: publicer mappen /docs på main.
   - Settings → Pages → Build and deployment → Source: "Deploy from a branch"
   - Branch: main
   - Folder: /docs  (IKKE "/" i roden af repo — vælg "/docs" i dropdown)
   - Save
   - Vent 1–2 min. Åbn: https://DIT_BRUGERNAVN.github.io/DIT_REPO/
   Kræver: docs/index.html findes (det gør den i dette projekt). Filnavnet skal
   være præcis "index.html" (små bogstaver).

3) METODE 1 (gh-pages-branch + workflow "Publish GitHub Pages")
   Kræver: Settings → Actions → General → Workflow permissions → Read and write.
   Pages → Source → Deploy from branch → gh-pages → /

4) METODE 2 (GitHub Actions artifact)
   Pages → Source → GitHub Actions.
   Workflow: "Deploy Pages (Actions-kilde)".
   Første gang: godkend miljøet "github-pages" under Actions, eller slå
   Required reviewers fra under Settings → Environments → github-pages.

5) BRUG KUN ÉN KILDE AD GANGEN
   Hvis du blander (fx Actions deploy mens Pages stadig peger på forkert mappe),
   får du 404 eller gammelt indhold.

6) ANDRE 404-ÅRSAGER (fra GitHub)
   - DNS hvis du bruger eget domæne
   - Browser-cache (prøv inkognito)
   - GitHub Status: https://www.githubstatus.com/
